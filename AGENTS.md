# AGENTS.md

## Purpose
このリポジトリでは、個人利用の iPhone 向け Web アプリを作成する。
Windows で開発し、GitHub で公開し、iPhone Safari で開いて使えることを前提とする。
PWA 化は必須。

## Tech Rules
- HTML / CSS / JavaScript のみで実装する
- React、Vite、TypeScript、Node.js、外部ビルドツールは禁止
- 外部HTTP API禁止
- サーバー使用は禁止
- フロントエンドのみで完結させる

## Layout Rules
- 画面幅は必ず iPhone 幅に合わせる
- 基本は縦スクロールのみ
- 横スクロールは原則禁止、明示指示時のみ許可
- iPhone Safari での表示を最優先に設計する
- タップしやすいボタンサイズにする
- 文字サイズは小さすぎないようにする
- スマホで片手操作しやすい構成を優先する

## Data Rules
- 入力データはブラウザ内に保存する
- localStorage または IndexedDB を使用する
- 更新処理を実行しても入力データは消えない構成にする

## PWA Rules
- manifest と Service Worker を必ず実装する
- GitHub 公開後に内容をアップデートしたとき、ユーザーが手動で最新版を読み込めるようにする
- 画面右上に更新アイコンを配置する
- 更新アイコンはかわいいデザインにする
- 更新アイコンはアイコンのみ表示し、説明文は付けない
- 更新アイコン押下時は次の確認文を表示する

「キャッシュを削除して最新版を読み込みます。入力データは消えません。実行しますか？」

- 実行時はキャッシュを削除して最新版を読み込む
- その際も入力データは保持すること

## Update Implementation Rules
- 更新ボタンは単なる再読み込みではなく、Service Worker の更新確認と静的アセットの再取得を行う
- registration.update() を利用して Service Worker の更新確認を行う
- キャッシュ名はバージョン管理し、更新時に古いキャッシュを削除する
- 必要に応じて skipWaiting と clients.claim を用い、新しい Service Worker を早く反映する
- app.js、styles.css などの更新が反映されやすいよう、キャッシュバスティングを考慮する
- 入力データ保存とアセットキャッシュは分離し、更新時も入力データを消さない

## Settings Rules
- 更新アイコンの右に設定アイコンを配置する
- 設定アイコンもかわいいデザインにする
- 設定アイコンはアイコンのみ表示し、説明文は付けない
- 設定アイコン押下でバックアップ画面を開く
- バックアップ画面には次の機能を持たせる
  - バックアップ文字列を作成
  - コピー
  - 貼付
  - 復元
- バックアップと復元は、ブラウザ保存データを文字列化して扱えるようにする

## UI Rules
- シンプルで見やすく、親しみやすいデザインにする
- 毎日使いやすい見た目を優先する
- アイコンはかわいく、ただし子どもっぽすぎない印象にする
- 更新アイコンと設定アイコンは右上に並べて配置する
- アイコンには文字ラベルを付けない

## Code Quality
- iPhone Safari で不安定になりやすい実装は避ける
- 複雑すぎる構造より、保守しやすく単純な構造を優先する
- 後から自分で修正しやすい、読みやすいコードにする
- 初心者でも追いやすい命名と構造にする

## Output Rules
- 必要ファイルをすべて作成する
- コードは省略せず全文を出力する
- ファイル名を明記する
- 実装時に採用した前提があれば明記する
- 最後に保存方法、配置方法、GitHub 公開時の注意点、iPhone での確認方法を説明する
